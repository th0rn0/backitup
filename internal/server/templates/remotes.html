Add Google Drive remote

Requires a Google Cloud project with the Drive API enabled and an OAuth 2.0 client (Web application type). The server redirects your browser to Google for consent, then exchanges the code for a token — no manual config file transfer needed.

{{if not .PublicAPI}}
BACKITUP_PUBLIC_API is not set. Set it to the server's public URL (e.g. https://backitup.example.com) before connecting Google Drive — it is used as the OAuth redirect URI.
{{end}}
How to create a Google OAuth 2.0 client ▸
  1. Open Google Cloud Console → APIs & Services → Library and enable the Google Drive API.
  2. Go to APIs & Services → Credentials → Create Credentials → OAuth 2.0 Client ID.
  3. Application type: Web application.
  4. Add an Authorised redirect URI: {{if .PublicAPI}}{{.PublicAPI}}{{else}}http://your-server{{end}}/oauth/gdrive/callback
  5. Copy the Client ID and Client Secret from the resulting dialog and paste them below.
Must match the client's Offsite destination value. The built-in dropdown uses gdrive.
